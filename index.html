Cervical cancer tumour segment­ation using deep learning

This was my Master's project conducted at the University of Manchester with collaboration with The Christie NHS Foundation Trust. The first part involved setting up a project pipeline to read in MRI image files (in NIfTI file format), pre-process them and storing these images. These were then used for large feature extraction using a package called radiomics.

The second part involved using the above pipeline, and a U-Net style convolutional neural network (CNN) to employ deep learning; this was used to segment two main defined cancerous regions - the gross tumour volume (GTV) which indicates the visible, macroscopic spread, and clinical tumour volume (CTV), which includes the GTV plus some of the surrounding area (containing possible microscopic spread).

GTV segmentation had also never done before using deep learning for cervical cancer at the time. The technique employed transfer learning using the ImageNet database (due to the limited size of the dataset of cervical cancer images) and in the end, accuracies of 55-80% were obtained in identifying the correct GTV/CTV regions. This project was predominantly Python-based though some MATLAB and Lua code had to be written as well. For future work, it would also be possible to repeat this technique on other sets of clinical images.

Marine mammal audio classifi­cation using deep learning

After conducting research, a suitable CNN architecture was found and then coded up in keras/tensorflow. The Watkins Mammal Database (containing around 40 mammals in total) with audio clips from different years were scraped from the site, collected, pre-processed and 6 types of audio features were then fed into a state-of-the-art model.

The model on the whole did very well with accuracies of 93% obtained in classifying between the different mammals in the test set. The model struggled with mammals which had the fewest audio clips (as the model did not have enough examples to learn enough features).

The same architecture was then tested for the Kaggle Right Whale Detection challenge that aimed to classify between sounds of right whales and other sounds (e.g. boats, propellers, other whales), achieving an accuracy of 85%; this was impressive as top-performing methods from the challenge required significantly more work with more manual processing, which this model did not require. The code was writen completely in Python.

Radio frequency signal classifi­cation using deep learning

Radio frequency (RF) signals can be described by I/Q data (which contains information on the amplitude and phase). In this project, a CNN and LSTM structure was developed in keras/tensorflow respectively to test out the classification performance.

Signals were of different modulations (QPSK, BPSK, 16QAM, 8PSK, DQPSK) and using their raw I/Q data, the models were trained. The CNN seemed to show far superior performance compared to the LSTM and the data used was synthetically generated using MATLAB (using the RF toolbox). The confusion matrices plotted showed that there were very high classification accuracies between the 5 signals with an average of around 95%.

Another idea explored was to see if the models could train on anomaly detection where one signal may be QPSK and another may be QPSK, mixed with another signal of lower intensity (an anomaly). The CNN showed superior performance to the LSTM again, and the resulting confusion matrices showed accuracies in the 92-99% region, depending on the signal pair. The code was written completely in Python, though some MATLAB was written for the synthetic signal generation.

Fictional University
